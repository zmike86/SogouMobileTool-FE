V1.6 ~ V2.0 
trunk还是1.6.1的版本，但这个版本最大的改动就是封装了更多的组件, mod/Mocha/mod/ 下有目前可复用的界面原件; 

下半年的主要任务是针对IE6-8的GC策略修复内存泄露的问题, 
并且勾画3.0版本框架蓝图。这里包括单元测试框架的引入和打包部署流程的优化

dojo1.6在设计模式和OO上给予了很大帮助, 但随着项目日益复杂, 强有力的续航能力略显不足, 追求极致感官性能的同时代码日趋复杂.
归其原因首先模块机制不完善, 1.7+AMD的实现过于老套, 有很多缺陷(比如chrome的本地文件安全策略不被支持);
其次, 为了弥补模块加载机制的不完善、打包工具需在源码上做改动, 不被推荐;
最后最重要一点, 思想到了高度但是实现很不优质; 1.9+版本还有待考验<
